Registration

GET
/bnet/auth
First, log in with Battle.net authorization. To do this, go to the following address: after logging in, you will receive a token that you must pass as query(token):
POST
/bnet/auth/userinfo?token={token}
As a result, you will receive the tagName of the user, which you then have to pass when executing the request:
POST
/user/register
To register a user, provide the following data: { tagName, region, email, password } - body JSON
tagName: obtained from authorization,
region: eu / us
If we find an Overwatch player with the given tagName, registration will be completed and you will receive the status: 200

Login

POST
/user/login
The main login router requires two parameters: { email, password } as a result of login you will receive the following data:
userInfo: {

_id: (this is the user's MongoDB user ID),
notifications: (these are the user's notifications),
friends: (this is the user's friends list),
messages: (these are the user's private messages),
posts: (these are user-created posts),

},
gameInfo: {

name: (this is the username),
lvl: (this is the user's lvl),
avatar: (this is the user's avatar),
frame: (this is the user's avatar border),
star: (these are the user's lvl asterisks),

},

token: (user token)

}

Posty * header must be provided: 'Authorization': 'Bearer {token}'

GET
/post/list
This router displays all existing posts, but to check a single post, run the router below.
GET
/post/{post_id}
As you can see, there is a { post_id } parameter to pass, this is the post id.
POST
/post/add
This router is responsible for creating a new post. Parameters to be passed through the form using the multipart/form-data method are: { name, desc, file }
name: is the name of the post,
desc: is the description of the post,
file: is the mp4 file
POST
/post/like/{post_id}
This router is responsible for liking the post whose id is passed using the { post_id } parameter.
POST
/post/comment/{post_id}
This router is responsible for adding comments to the post, whose id is passed as the { post_id } parameter.
DELETE
/post/remove/{post_id}
Router responsible for removing a given post, whose id is passed as { post_id } parameter.

Friends * header must be provided: 'Authorization': 'Bearer {token}'

GET
/friends/list
By making a query to this router, you will receive a list of the user's friends, whose id is taken from the token.
POST
/friends/invite/{profile_id}
This router is responsible for sending invitations to the person whose id is passed as the { profile_id } parameter.
POST
/friends/accept/{invite_id}
This router is responsible for accepting the invitation, the id of which is passed as the { invite_id } parameter.
POST
/friends/remove/{invite_id}
This router is responsible for rejecting the invitation, the id of which is passed as the { invite_id } parameter.

Messages * header must be provided: 'Authorization': 'Bearer {token}'
